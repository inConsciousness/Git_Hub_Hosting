Deploying a Website on GitHub Pages

1. Prepare Your Repository

1.1 Create a GitHub Repository (if not already created)

  • Go to GitHub and sign in.
  • Click on the "+" icon in the top right corner and select "New repository."
  • Fill in the repository name, description, and other details.
  • Click "Create repository."

1.2 Clone the Repository (if you haven't already)

git clone https://github.com/yourusername/repositoryname.git

Replace yourusername and repositoryname with your GitHub username and repository name.

2. Add Your Website Files

2.1 Navigate to Your Project Directory

cd path/to/your/repository

2.2 Add Your Website Files

  • Ensure your main file is named index.html and placed in the root directory.
  • Add any other necessary files (e.g., CSS, JavaScript, images).

2.3 Commit Your Changes

git add .
git commit -m "Add website files"
git push origin master

Replace master with main if that is your branch name.

3. Configure GitHub Pages

3.1 Go to Your Repository on GitHub

  • Navigate to the repository where you have pushed your website files.

3.2 Open Settings

  • Click on the "Settings" tab at the top of the repository page.

3.3 Locate GitHub Pages Settings

  • Scroll down to the "Pages" section on the left sidebar or find the "Pages" option under the settings.

3.4 Set the Source

  • Branch: Select the branch you used for GitHub Pages (e.g., master or main).
  • Folder: Choose / (root) if index.html is in the root directory. This ensures that GitHub Pages will serve index.html as the homepage.

3.5 Save Changes

  • Click "Save" to apply the changes.
4. Verify Your Site

4.1 Find the URL

  • GitHub Pages will provide a URL where your site will be hosted. It typically looks like https://yourusername.github.io/repositoryname.

4.2 Access Your Site

  • Visit the URL to see your live site. It may take a few minutes for changes to appear.
Additional Notes
  • Troubleshooting: If your site does not appear as expected, check for errors in the browser console or network tab.
  • Custom Domains: If you have a custom domain, you can configure it in the GitHub Pages settings under the "Custom domain" section.
